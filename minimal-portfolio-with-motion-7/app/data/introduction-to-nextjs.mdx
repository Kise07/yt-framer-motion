# Introduction to Next.js

Next.js is a React framework created by Vercel that enables developers to build production-ready web applications with minimal configuration. While React provides the foundation for building user interfaces, Next.js adds the structure, conventions, and optimizations needed for real-world applications.

## Why Next.js?

React alone is a library, not a framework. It leaves architectural decisions — routing, data fetching, rendering strategy — entirely to the developer. Next.js solves this by providing sensible defaults and built-in features that would otherwise require dozens of third-party libraries.

## Key Features

**Server-Side Rendering (SSR)** — Pages are rendered on the server and sent as HTML to the browser, improving initial load performance and SEO.

**Static Site Generation (SSG)** — Pages can be pre-built at build time, resulting in instant load times for content that doesn't change frequently.

**File-Based Routing** — Creating a new page is as simple as adding a file to the `app/` directory. No router configuration needed.

**API Routes** — Build your backend directly inside your Next.js project using Route Handlers.

**Built-in Optimizations** — Automatic image optimization, font optimization, and script loading strategies come out of the box.

## App Router

Next.js 13+ introduced the App Router, which uses React Server Components and nested layouts. This architecture enables streaming, parallel routes, and improved data fetching patterns.

## When to Use Next.js

Choose Next.js when you need SEO, performance, and a structured project without configuring build tools, routing, and rendering strategies from scratch. It is ideal for blogs, portfolios, e-commerce sites, and dashboards.
